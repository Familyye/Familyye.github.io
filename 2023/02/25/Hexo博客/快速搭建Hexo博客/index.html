1. Node.js的安装

  1. 在Node官网下载与自己系统相配的Node的安装程序，否则会出现安装问题，我个人的版本是12.19.0，目前版本已经更新到19.7.0，建议选个低一些的版本，否则后面会出现各种不兼容的问题，例如系统无法识别。历史版本下载地址：https://nodejs.org/en/download/releases/
  2. 软件建议下载后缀名为msi，例如node-v12.19.0-x64.msi，一路点击next即可。建议选择默认路径C:/Program Files/nodejs/，如果想切换其他盘或者把环境遍历切换到自定义路径也可以，但是这里的坑很多
  3. 安装完成后可以WIN+R键，输入cmd，打开命令行窗口，执行node -v命令，在这里就可以看到版本信息，同时也证明你安装成功了
  4. 后续需要npm下载各种模块，默认是从国处服务器下载，速度较慢，建议配置成淘宝镜像。在命令行窗口(cmd)中执行以下命令

  1. 在Git官网下载任一版本即可，有时官网下载会很慢，可以缓存淘宝的开源镜像下载
  2. 跟Node.js不同，Git的安装没有多大要求，一路点击next即可，可以在路径安装时更换一个路径
  3. 配置用户名和邮箱，具体命令如下
  4. 通过git config -l可以检查是否配置成功
  5. Git的一些知识
  • Git CMD是windows命令行的指令风格
  • Git Bash是linux系统的指令风格（建议使用）
  • Git GUI是图形化界面（新手学习不建议使用）

  4. 博客目录有以下文件
  • node_modules：依赖包
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • .gitignore：发布时忽略的文件（可忽略）
  • _config.landscape.yml：主题的配置文件
  • config.yml：博客的配置文件
  • package.json：项目名称、描述、版本、运行和开发等信息

4.1 创建仓库

在Github官网个人主页，点击右上角的+按钮，选择New repository，创建一个<用户名>.github.io(必须)的仓库。

5. 将静态博客挂载到GitHub Pages

  1. 在Git Bash中安装hexo-deployer-git
  2. 修改Hexo博客的_config.yml文件，在里面可以修改大部分配置，详细可参考官方配置描述。
  3. 修改好配置后，运行如下命令，将代码部署到 GitHub（Hexo三连）。
  • hexo clean：删除之前生成的文件，若未生成过静态文件，可忽略此命令。
  • hexo generate：生成静态文章，可以用hexo g缩写
  • hexo deploy：部署文章，可以用hexo d缩写。这时可能要你输入username和password
  1. 如果出现Deploy done，则说明部署成功了。稍等两分钟，打开浏览器访问：https://blhorizon.github.io ，这时候我们就可以看到博客内容了。

5. 文本编辑器VSCODE

5.1 VSCODE不能在终端执行程序脚本

  1. 右击vscode图标，选择以管理员身份运行；
  2. 在vscode终端中执行get-ExecutionPolicy，显示Restricted，状态是禁止的；
  3. 执行set-ExecutionPolicy RemoteSigned；
  4. 再执行get-ExecutionPolicy，显示RemoteSigned，状态解禁，可以运行。
